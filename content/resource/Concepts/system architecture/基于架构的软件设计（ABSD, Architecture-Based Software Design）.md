## 概述

### 定义

基于架构的软件设计(Achitecture-Based Software Design，ABSD)方法有三个基础，分别是对`系统进行功能分解`、`采用架构风格`实现质量属性与商业需求、`采用软件模板`设计软件结构。

### 核心概念

ABSD方法主要包括架构需求等6个主要活动

其中架构复审活动的目标是标识潜在的风险，及早发现架构设计中的缺陷和错误；架构演化活动针对用户的需求变化，修改应用架构，满足新的需求。
